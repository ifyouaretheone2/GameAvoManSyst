## Why

当前系统仅有预置管理员账号与登录接口，无法自助开通账号，也无法在运行中把普通账号提升为管理员；站点默认进入公开首页，与“先登录再使用后台”的预期不一致。需要在不破坏现有内容管理能力的前提下，补齐注册、角色提升与默认入口体验。

## What Changes

- 提供面向普通用户的自助注册能力（账号、密码等必要字段，服务端校验与持久化）。
- 在数据模型与 JWT 中区分普通用户与管理员；仅管理员可访问现有后台管理能力。
- 管理员可将指定普通用户提升为管理员（幂等、不可自我降级等边界需明确）。
- 将应用初始路由（根路径或默认落地页）调整为登录页；已登录用户按角色跳转到合适首页（如后台仪表盘）。
- 前端补充注册页/流程，并在管理后台提供“提升为管理员”的入口或 API 对接。

## Capabilities

### New Capabilities

- `auth-user-registration`: 自助注册、密码存储策略、与登录接口的衔接；注册后默认非管理员角色。
- `auth-default-route-login`: 路由与导航：未登录访问根路径进入登录页；登录成功后的默认重定向规则。
- `admin-role-promotion`: 仅管理员调用：将普通用户提升为管理员；权限校验、错误码与安全约束。

### Modified Capabilities

（无：现有 `openspec/specs/` 下能力包与本次认证/角色变更无直接需求重叠。）

## Impact

- 后端：`adminController` / `adminRoutes`、认证中间件、数据库表结构或新表、JWT payload 扩展、种子数据与迁移脚本。
- 前端：`router/index.js` 默认路由与守卫、`Login.vue`、新增 `Register.vue`（或等价视图）、`stores/user.js`、`api` 层；管理后台需列表/操作以提升角色（若尚无用户管理页则需新增最小界面）。
- 运维/配置：环境变量说明（若引入注册开关、初始管理员等）。
